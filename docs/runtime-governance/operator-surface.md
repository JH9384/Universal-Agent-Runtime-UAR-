# Operator Surface Guide

## Objective

Provide a structured operational surface for runtime visibility, replay continuity inspection, governance diagnostics, and runtime operational interaction.

## Surface Domains

- runtime health
- runtime telemetry
- replay continuity
- replay restoration visibility
- governance diagnostics
- topology visibility
- operational event streams

## Operator Contracts

The operator surface is built from:

- operator surface cards
- replay timeline views
- runtime API clients
- telemetry streams

## Strategic Direction

The operator surface transitions UAR from deployable infrastructure into operable runtime infrastructure.
