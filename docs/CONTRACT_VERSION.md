# Contract Version Policy

## Objective

Provide stable evolution rules for runtime contracts, replay payloads, telemetry structures, and operator-facing runtime schemas.

## Contract Domains

- runtime events
- replay payloads
- operator contracts
- topology payloads
- telemetry payloads
- governance payloads

## Versioning Rule

Breaking changes require:

1. explicit version increment
2. compatibility review
3. migration notes
4. replay continuity review

## Strategic Direction

Stable contracts are required for long-lived runtime continuity and operator-platform evolution.
