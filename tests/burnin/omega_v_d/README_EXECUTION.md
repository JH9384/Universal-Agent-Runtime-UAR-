# Ω-V.D Burn-In Execution Checklist

## Pre-Run

- [ ] topology mutation enabled
- [ ] broadcast propagation enabled
- [ ] governance dampening enabled
- [ ] replay convergence enabled
- [ ] restoration equilibrium enabled
- [ ] frontend synchronization enabled

## Runtime Validation

Observe continuously during burn-in:

- queue pressure
- topology stability
- synchronization entropy
- governance pressure
- restoration confidence
- feedback dampening
- convergence score

## Failure Escalation Rules

Immediately stop burn-in if:

- memory growth trend appears
- propagation runaway appears
- topology oscillation appears
- governance recursion amplifies
- replay fragmentation accelerates
- restoration backlog grows unbounded
- frontend render instability appears

## Post-Run

- capture convergence metrics
- update certification report
- document tuning changes
- update KNOWN_STABILITY_LIMITS.md
- determine release readiness
